    Overview

    Introduction

    Bumblebee API (Application Programming Interface) is built around REST.

    Bumblebee API returns JSON-encoded responses and uses standard HTTP response codes and authentication

    Each endpoint has full description with sample responses

    Please go through the overview section before digging deeper into core resources.

    Base URL: https://iot.bumblebeeai.io/api/

    Headers

    When consuming Bumblebee API, ensure you include the below headers in every request

    Accept: application/json Content-Type: application/json

    Authentication

    Bumblebee API uses Access Tokens to authenticate requests.

    Bumblebee API uses OAuth2.0 Code Grant Authorization to enable your application to obtain Access Token. For detailed guide, please visit OAuth 2.0 Code Grant Flow

    Authorization is required in all API requests and is added as a header

    Authorization: Bearer meowmeowmeow

    Note: The word Bearer in Authorization is case sensitive, using bearer will result in authorization error

    Access Tokens carry many privileges, so be sure to keep them secure! Do not share your secret Access Token in publicly accessible areas such as public documents, client-side code, and so forth.

    You must replace Bearer meowmeowmeow with your personal API key.

    Authentication

    Login

    Login

    The request should include the following parameters:

    POST /auth/login

    Name Type Required Description
    phone string Yes Required without email input Unique mobile number of the the user, it must contains the country key ex: +201000000000.
    email string Yes Required without phone input Unique email of the the user.
    password string true The password of the user. min: 8 characters
    role string true The type of user must be in unified_engineer, driver, contractor

    Example responses

    200 Response

    {
    "message": "Success.",
    "data": {
        "name": "Hisham Magdy",
        "token_type": "Bearer ",
        "access_token": "eyJ0eXAiOiJKV1QiLC...",
    }
}

    401 Response

    {
    "message": "Unauthenticated.",
}

    Forgot Password

    Forgot Password

    The request should include the following parameters:

    POST /auth/forgot_password

    Name Type Required Description
    phone string true Required without email input Unique mobile number of the the user, it must contains the country key ex: +201000000000.
    email string true Required without phone input Unique email of the the user.
    password string true The password of the user. min: 8 characters
    role string true The type of user must be in unified_engineer, driver, contractor

    Example responses

    200 Response

    {
    "message": "Success.",
}

    402 Response

    {
    "message": "Unauthenticated.",
}

    422 Response

    {
    "message": "The given data was invalid.",
    "errors": {
        "role": [
            "The role field is required."
        ],
        "email": [
            "The selected email is invalid."
        ]
    }
}

    Update Password

    Update Password

    POST /auth/update_password

    Description

    You can use this API to reset password in forgot password or updating password in profile

    The request should include the following parameters:

    Name Type Required Description
    code string true Required without old password input The verification code.
    old_password string true Required without code input The old password of the user.
    password string true The new password of the user.

    Example responses

    200 Response

    {
    "message": "Success."
}

    401 Response

    {
    "message": "Unauthenticated.",
}

    422 Response

    {
    "message": "The given data was invalid.",
    "errors": {
        "old_password": [
            "The old_password field is required."
        ],
        "password": [
            "The password field is invalid."
        ],
        "code": [
            "The code field is required."
        ]
    }
}

    Update Job

    Update Job

    POST /unified/jobs/{id}

    The request should include the following parameters:

    Name Type Required Description
    status string true Required without feedback inputs The new status of the job.
    job_type_id integer true Required without status input The ID of the service type.
    expenses array false The expenses of the job.
    expenses_receipt file false The expenses file of the job.
    job_images array false The job images of the job. The array must contains files

    The expenses array should include the following parameters:

    Name Type Required Description
    name string true The name of the expenses.
    cost string true The actual cost of the expenses.
    comment string false The comment of the expenses.

    Example responses

    200 Response

    {
    "data": {
      "message": "Success."
    }
}

    401 Response

    {
    "message": "Unauthenticated.",
}

    422 Response

    {
    "message": "The given data was invalid.",
    "erorrs": {
      "expenses_receipt": [
          "The expenses must be an array."
      ],
      "service_id": [
          "The service id was invalid."
      ]
    }
}

    Errors

    Bumblebee API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The api requested is hidden for administrators only.
    404 Not Found -- The specified API could not be found.
    405 Method Not Allowed -- You tried to access a API with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The API requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
